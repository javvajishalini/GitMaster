# Contributing to GitMaster

First off, thank you for considering contributing to GitMaster! It's people like you that make GitMaster such a great learning tool for developers.

We welcome all contributions, including bug reports, bug fixes, feature requests, documentation improvements, and UI enhancements.

## Getting Started

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** to your local machine:
   ```bash
   git clone https://github.com/YOUR-USERNAME/GitMaster.git
   ```
3. **Navigate to the project directory**:
   ```bash
   cd GitMaster
   ```
4. **Install dependencies**:
   ```bash
   npm install
   ```
5. **Start the development server**:
   ```bash
   npm run dev
   ```

## Workflow for Making Changes

1. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-bug-fix
   ```
2. **Make your changes**. Please ensure your code follows the existing style and conventions.
3. **Test your changes** locally in the browser.
4. **Commit your changes** with a clear and descriptive commit message:
   ```bash
   git commit -m "Add short description of your change"
   ```
5. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request** against the `main` branch of the original GitMaster repository. Provide a clear description of the problem you've resolved or the feature you've added.

## GSSoC Contributors

If you are contributing as part of the **GirlScript Summer of Code (GSSoC)**:
- Please check the Issues tab for issues labeled with `gssoc`, `good first issue`, or `help wanted`.
- Comment on the issue you want to work on to get it assigned to you before starting.
- Make sure to follow the Code of Conduct and be respectful to all community members.

## Code Style

- This project uses React (Vite) and Tailwind CSS v4.
- Prefer functional components and React Hooks.
- Ensure your UI changes are responsive (test on mobile and desktop viewports).
- Use descriptive variable and function names.

Thank you for contributing!
